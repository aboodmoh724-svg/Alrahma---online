# خطة الأرشيف التعليمي وقاعدة البيانات التاريخية

هذه الوثيقة هي مرجع العمل على مشروع أرشفة بيانات الرحمة التعليمية والتاريخية. الهدف منها منع التداخل بين المحادثات والتعديلات، وتثبيت طريقة آمنة ومنظمة للتعامل مع بيانات سنوات طويلة من Google Drive والجداول والنماذج والملفات.

## الهدف

بناء قاعدة بيانات تعليمية وإدارية دقيقة، لا تعتمد على نقل كل الملفات كما هي، بل على:

- جرد الملفات والمجلدات.
- استخراج البيانات المفيدة.
- تنظيف الأسماء والأرقام والعلاقات.
- مطابقة الطلاب والمعلمين والحلقات عبر السنوات.
- ربط الملفات الأصلية كمراجع عند الحاجة.
- عرض النتائج في المنصة بطريقة قابلة للبحث والمراجعة.

## المبدأ العام

لا يتم اعتماد أي بيانات حساسة أو دمج طالبين أو حذف معلومات أو تصنيف حالة اجتماعية بشكل نهائي اعتمادا على الذكاء الاصطناعي وحده.

النظام يقترح، والإدارة تراجع وتعتمد.

## ما يدخل قاعدة البيانات

تدخل قاعدة البيانات البيانات المنقحة والمفيدة، مثل:

- الطلاب وأسماؤهم المعتمدة.
- المعلمون والمشرفون ومناصبهم.
- الحلقات والمسارات والسنوات.
- أرقام أولياء الأمور ووسائل التواصل.
- تقارير الحفظ والمراجعة والتقييم.
- حضور وغياب عند توفر مصدر موثوق.
- محاضر الاجتماعات والقرارات.
- روابط الملفات المهمة المرتبطة بسجل طالب أو معلم أو حلقة.

## ما لا يدخل كملف خام

لا ترفع الملفات الكبيرة إلى سيرفر المنصة مباشرة، خصوصا:

- الفيديوهات الإعلامية.
- الصور الضخمة.
- أرشيفات Drive الكبيرة.
- الملفات المكررة أو غير المصنفة.

هذه تبقى في Google Drive أو تخزين خارجي، وتدخل المنصة فقط كفهرس يحتوي الرابط والوصف والتصنيف.

## مستويات السرية

يجب التعامل مع البيانات حسب مستوى حساسيتها:

- عام داخلي: بيانات تعليمية عادية.
- إداري: ملفات الحلقات والمعلمين والمتابعة.
- حساس: أرقام، وثائق، حالات اجتماعية، أيتام، صور هويات.
- حساس جدا: أي بيانات مالية أو حالات اجتماعية تفصيلية أو وثائق شخصية.

أي قسم حساس يحتاج صلاحيات منفصلة وسجل تدقيق لاحقا.

## مراحل التنفيذ

### المرحلة 1: عينة اختبار

نبدأ بسنة واحدة أو حلقة واحدة، وليس كامل الأرشيف.

المطلوب في العينة:

- ملف الطلاب أو الردود بصيغة Excel أو Google Sheet.
- قائمة ملفات Drive إن وجدت.
- 10 إلى 30 ملفا متنوعا: PDF، Word، صور، تقارير.
- وصف مختصر للسنة أو الحلقة أو القسم.

النتيجة المتوقعة:

- جدول أولي للطلاب والمعلمين والحلقات.
- قائمة ملفات مصنفة.
- اقتراحات مطابقة أسماء.
- قائمة عناصر تحتاج مراجعة.
- نموذج حقول مقترح للقاعدة.

### المرحلة 2: بناء نموذج الاستيراد

بعد نجاح العينة، يتم بناء آلية استيراد مبدئية:

- قراءة الجداول المنظمة.
- تصنيف الملفات.
- ربط الملفات بالطلاب أو المعلمين إن أمكن.
- عرض نتائج "مؤكد / محتمل / يحتاج مراجعة".
- عدم اعتماد النتائج تلقائيا.

### المرحلة 3: التوسيع سنة سنة

يتم العمل على دفعات واضحة:

- سنة محددة.
- قسم محدد.
- حلقة أو عدة حلقات.
- دفعة ملفات محدودة.

كل دفعة تنتهي بتقرير مراجعة قبل الانتقال للدفعة التالية.

### المرحلة 4: البحث والعرض

بعد توفر بيانات كافية، يتم بناء واجهات للبحث:

- ملف الطالب التاريخي.
- ملف المعلم التاريخي.
- سجل الحلقة عبر السنوات.
- تقارير سنة محددة.
- محاضر وقرارات مرتبطة.
- ملفات ومراجع مرتبطة.

### المرحلة 5: الأرشيف الإعلامي

الأرشيف الإعلامي يؤجل إلى مرحلة لاحقة.

عند تنفيذه لا يتم رفع الفيديوهات الضخمة إلى السيرفر. يتم فهرستها فقط:

- العنوان.
- الرابط.
- السنة.
- المناسبة.
- النوع.
- القسم أو الحلقة.
- قابل للنشر أو داخلي.
- كلمات مفتاحية.

## أفضل صيغ التسليم

الأفضل حسب الأولوية:

1. Google Sheet أو Excel للبيانات المنظمة.
2. Google Forms responses مربوطة بملف Sheet.
3. CSV لقائمة ملفات Drive.
4. Word أو Google Docs للوثائق النصية.
5. PDF نصي.
6. PDF مصور أو صور، وتكون نتائجها بحاجة مراجعة.

## صيغة دفعة العمل

عند إرسال دفعة جديدة، يفضل أن تبدأ الرسالة بهذا الشكل:

```text
المسار: قاعدة بيانات الأرشيف التعليمي
الدفعة: سنة 2025 / حلقة كذا / قسم كذا
المرفقات:
- ملف الطلاب
- قائمة الملفات
- ملفات تقارير

الهدف من الدفعة:
- استخراج الطلاب
- مطابقة المعلمين
- تصنيف الملفات
- تحديد ما يحتاج مراجعة
```

## قواعد السلامة

- لا يتم تعديل بيانات مالية أو حساسة إلا بطلب صريح.
- لا يتم حذف ملفات أصلية.
- لا يتم دمج سجلات متشابهة نهائيا دون مراجعة.
- لا يتم اعتماد بيانات OCR من الصور دون إشارة أنها تحتاج مراجعة.
- لا يتم نقل ملفات Drive الضخمة إلى السيرفر.
- كل تغيير في قاعدة البيانات يجب أن يكون له migration واضح عند الحاجة.

## حالة المشروع

الحالة الحالية: تخطيط وتجهيز منهجية العمل.

الخطوة التالية: استلام عينة صغيرة من سنة أو حلقة واحدة، ثم إنتاج نموذج تصنيف ومطابقة أولي.
